## 酒店客房管理系統
### 需求說明書
#### 
1. 文件目的

本文件旨在完整說明「酒店客房管理系統」之系統需求，作為系統分析、設計、開發、測試與驗收的主要依據，確保系統功能符合實際飯店營運需求，並降低開發過程中溝通錯誤的風險。

2. 系統背景說明

傳統飯店多以人工或簡易試算表管理客房狀態，容易產生資料錯誤與即時性不足的問題。因此，本系統透過資訊化方式，集中管理客房、入住資訊與人員操作，以提升服務效率與管理品質。

3. 系統使用者角色

櫃檯人員：
查詢即時房態、辦理顧客入住與退房、建立與修改顧客基本資料

管理者：
新增與管理房型資料、設定房價與折扣、查閱營運統計與報表

清潔人員：
查看需清潔房間清單、回報清潔完成狀態

4. 功能性需求
客房基本資料管理（房號、樓層、房型、價格）
客房狀態管理（空房、已入住、清潔中、維修中）
入住管理（入住登記、入住日期、退房日期）
退房管理（費用結算、房態更新）
使用者帳號與權限管理
查詢與報表輸出功能

5. 非功能性需求
系統操作回應時間需低於 3 秒
系統需具備基本資安機制（登入驗證、權限控管）
系統需支援至少 10 人同時操作
資料須每日自動備份

6. 使用者介面需求
採用 Web 介面，無需安裝軟體
操作畫面簡潔，適合非資訊背景人員使用
提供圖示化房態顯示

### 概要設計說明書
#### 
1. 系統整體架構

本系統採用三層式架構：
表示層（Presentation Layer）：使用者操作介面
應用層（Application Layer）：處理系統邏輯與流程
資料層（Data Layer）：負責資料儲存與存取

2. 系統模組劃分
使用者與權限管理模組
客房資料管理模組
入住與退房管理模組
清潔狀態管理模組
報表與統計模組

3. 模組關聯說明
各模組透過後端 API 進行資料交換，避免模組間高度相依，提高系統維護性與擴充性。

### 詳細設計說明書
#### 
1. 客房管理模組詳細設計
資料表設計（範例）：
room_id（房號）
room_type（房型）
price（價格）
status（房態）

2. 主要功能：
新增客房資料
修改客房資訊
刪除客房資料
查詢客房狀態

3. 入住管理流程設計
櫃檯人員登入系統
系統顯示所有空房
選擇房間並輸入顧客資料
系統確認後更新房態為「已入住」

4. 系統邏輯說明
系統僅允許「空房」狀態之房間進行入住
退房完成後，自動將房態轉為「清潔中」

### 測試計畫
#### 
1. 測試目的
確保系統功能符合需求文件規範，並於實際使用環境下穩定運作。

2. 測試項目
功能測試：各項操作是否正確
介面測試：操作是否直覺
權限測試：不同角色功能限制
效能測試：多人同時使用狀況

3. 測試方法
黑箱測試（依功能操作）
情境測試（模擬實際飯店流程）

4. 測試資源與時程
測試人員：1–2 人
測試環境：測試伺服器與測試資料庫
測試期間：約 1–2 週
