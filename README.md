1. 文件目的

本文件的主要目的在於說明「酒店客房管理系統」之整體規劃與設計內容，作為系統開發、實作與後續維護的重要依據。透過本文件，能讓開發人員、使用者以及相關管理人員，對系統的功能需求、操作流程與設計方向有一致且清楚的認知，避免在開發過程中產生理解落差。

本文件詳細描述系統所需具備的核心功能，包含客房狀態管理、訂房管理、入住與退房流程，以及基本的資料管理機制，以協助飯店人員能更有效率地進行日常作業。同時，也說明系統在操作介面與使用體驗上的設計原則，確保使用者能快速上手，降低學習與操作錯誤的機率。

此外，本文件也作為系統測試與驗收的重要參考依據，透過事前明確定義功能需求與流程，讓後續的測試能依照文件內容進行檢核，確保系統運作符合預期目標。對於專題製作而言，本文件不僅能幫助我們在開發過程中有明確方向，也能展現專題的完整性與專業度，讓整個系統設計更具結構性與實用價值。

2. 系統背景說明


隨著科技快速發展，服務業逐漸導入資訊系統來提升管理效率，飯店與旅宿業也不例外。傳統的客房管理多半仰賴人工登記或紙本記錄，不僅容易發生資料遺漏或錯誤，也會在尖峰時段增加櫃檯人員的工作負擔，影響整體服務品質。因此，開發一套完善且操作直覺的酒店客房管理系統，成為現代飯店經營中相當重要的一環。

本專題以「酒店客房管理系統」為主題，目的是透過系統化的方式整合訂房、入住、退房與客房狀態管理等流程，讓管理人員能即時掌握每一間房間的使用情況，例如空房、入住中、清潔中或已預訂等狀態，減少人工作業所造成的錯誤。同時，也能加快櫃檯處理速度，提高顧客入住與退房時的整體體驗。

在製作本系統的過程中，我們觀察到實際飯店作業流程相當複雜，若沒有清楚的系統規劃，容易造成資訊混亂。因此，我們透過系統分析與設計的方式，將實際作業流程轉換成系統功能，並搭配清楚的介面設計，讓使用者能快速理解並操作。本專題希望藉由此系統，展現資訊科技在日常生活中的實際應用，也提升我們對系統開發流程的理解與實作能力。

3. 系統使用者角色

櫃檯人員：
櫃檯人員是酒店客房管理系統中最主要的使用者之一，負責與顧客直接接觸，並處理每日最頻繁的系統操作。透過本系統，櫃檯人員可以即時查詢所有客房的狀態，例如空房、已入住、清潔中或已預訂，避免因資訊不即時而造成重複訂房或安排錯誤。當顧客辦理入住時，櫃檯人員可快速完成入住登記，並建立顧客的基本資料，如姓名、聯絡方式與入住期間；退房時，也能透過系統完成結帳與狀態更新，大幅提升作業效率。

此外，系統也支援顧客資料的建立與修改，讓櫃檯人員能隨時更新顧客資訊，減少紙本紀錄帶來的錯誤與遺漏。透過系統化管理，不僅能縮短顧客等待時間，也能讓櫃檯作業流程更加順暢與專業。

管理者：
管理者主要負責整體酒店營運與系統設定，是系統中具有較高權限的角色。透過酒店客房管理系統，管理者可以新增與管理不同房型資料，例如標準房、豪華房或家庭房，並依照實際需求設定房價與折扣方案，使價格管理更加彈性與有系統。同時，管理者也能依照淡旺季或促銷活動，快速調整房價，提高營運效率。

此外，系統提供營收統計與報表查詢功能，讓管理者能清楚掌握每日、每月或特定期間的住房率與營收狀況，作為經營決策的重要參考。透過視覺化的報表呈現，管理者不需人工計算即可快速了解經營成果，減少錯誤並節省時間。整體而言，管理者透過此系統能更有效地掌控飯店營運狀況，提升管理品質與決策效率。

清潔人員：
清潔人員在酒店營運中扮演著維持住宿品質的重要角色，而透過酒店客房管理系統，可以讓清潔工作更加有條理與效率。清潔人員可透過系統查看目前需要清潔的房間清單，清楚知道哪些房間已退房、哪些正在清潔中，避免重複或遺漏工作。這樣的設計能有效降低人為溝通錯誤，讓清潔流程更加順暢。

當清潔完成後，清潔人員只需在系統中回饋清潔完成狀態，系統便會即時更新該房間的狀態，讓櫃檯人員與管理者能第一時間掌握房況。透過這樣的即時回報機制，不僅能縮短房間重新開放的時間，也能提升整體客房周轉率。此系統讓清潔人員的工作流程更清楚，也強化了部門之間的協作效率。

4. 功能性需求

本酒店客房管理系統的功能需求主要以提升飯店作業效率與管理準確性為目標，透過系統化方式整合客房、顧客與人員作業流程。系統需提供即時客房狀態查詢功能，讓櫃檯人員能快速掌握各房間目前狀態，如空房、已入住、清潔中、已預訂或暫停使用，避免重複安排或判斷錯誤。

在顧客管理方面，系統須支援顧客入住與退房作業，包含建立、修改與查詢顧客基本資料，並能在退房時自動更新房間狀態，使資訊保持一致性。同時，系統應提供訂房與房型管理功能，讓管理者能新增與維護不同房型資料，並設定房價與相關折扣，以因應不同季節與促銷需求。

此外，系統需支援清潔人員作業需求，提供需清潔房間清單與清潔狀態回報功能，讓房務進度能即時同步給櫃檯與管理者，提升客房周轉效率。在管理層面，系統應具備營收統計與報表查詢功能，協助管理者快速掌握經營狀況，作為決策參考。整體而言，本系統的功能需求著重於資訊即時性、操作便利性與跨角色協作，確保酒店日常營運能更加順暢且有效率。

5. 非功能性需求

本酒店客房管理系統除了具備完整的功能需求外，也必須在非功能性需求上達到一定水準，才能確保系統在實際使用時穩定、安全且方便操作。首先在效能需求方面，系統需能即時回應使用者操作，例如櫃檯人員查詢客房狀態或辦理入住時，畫面應在短時間內完成更新，避免因延遲影響服務品質。

在可用性與易用性方面，系統介面設計需清楚直覺，讓不同角色的使用者，如櫃檯人員、管理者與清潔人員，都能快速上手操作，減少學習成本與操作錯誤發生的機率。

此外，安全性需求也是重要的一環，系統需具備帳號登入與權限控管機制，確保不同使用者只能存取符合其身分的功能與資料，避免顧客個資或營收資料外洩。

最後，在可靠性與維護性方面，系統應具備穩定運作能力，並能方便進行後續功能擴充與維護，以因應飯店未來營運規模成長的需求。

6. 使用者介面需求

介面簡潔、操作直覺
系統整體介面設計需以簡單明瞭為原則，常用功能應清楚標示，避免過多複雜選單。櫃檯人員能在短時間內完成入住或退房操作，減少因操作困難造成的服務延誤。

角色導向介面顯示
系統應依不同使用者角色（櫃檯人員、管理者、清潔人員）顯示對應的功能畫面。登入後只顯示該角色可使用的功能，降低誤操作風險，也提升使用效率。

即時狀態視覺化呈現
客房狀態（空房、入住中、待清潔、維修中）需以顏色或圖示清楚標示，讓使用者一眼就能判斷目前狀況，提升工作判斷速度。

資料輸入與提示機制
系統在輸入資料時，需提供欄位提示與錯誤提醒，例如未填寫必要資料時跳出警示，避免資料不完整或輸入錯誤。

跨裝置與解析度支援
介面需支援電腦與平板等不同裝置，確保在不同解析度下仍能正常顯示，方便清潔人員或管理者於現場操作。

操作回饋明確
使用者進行操作後，系統應即時顯示成功或失敗訊息，如「入住完成」或「資料儲存成功」，讓使用者清楚知道目前操作狀態。

### 概要設計說明書
#### 
1. 系統整體架構

本系統採用三層式架構：
表示層（Presentation Layer）：使用者操作介面
應用層（Application Layer）：處理系統邏輯與流程
資料層（Data Layer）：負責資料儲存與存取

2. 系統模組劃分
使用者與權限管理模組
客房資料管理模組
入住與退房管理模組
清潔狀態管理模組
報表與統計模組

3. 模組關聯說明
各模組透過後端 API 進行資料交換，避免模組間高度相依，提高系統維護性與擴充性。

####

status（房態）

2. 主要功能：
新增客房資料
修改客房資訊
刪除客房資料
查詢客房狀態

3. 入住管理流程設計
櫃檯人員登入系統
系統顯示所有空房
選擇房間並輸入顧客資料

系統僅允許「空房」狀態之房間進行入住
退房完成後，自動將房態轉為「清潔中」

### 心得感想

在這次以「酒店客房管理系統」為主題的專題製作中，我們從一開始對系統運作只有模糊的概念，到最後能夠完整說明系統功能、流程與架構，對我們來說是一段收穫很多的學習過程。以前住飯店時，只會覺得櫃檯安排房間很快，但實際深入了解後，才發現背後需要一套完善的管理系統來支援整個流程。

在報告製作過程中，我們參考了酒店客房管理系統的介面示意圖，了解櫃檯人員如何即時掌握房態，例如空房、已入住、清潔中或已退房等狀態。這些資訊如果沒有清楚呈現，很容易造成操作錯誤，因此我們在設計與整理圖片時，特別注意顏色區分與版面配置，讓使用者能快速理解目前狀況。

在系統架構設計方面，我們學習到三層式架構的概念，包含表示層、商業邏輯層與資料存取層。一開始對這些名詞並不熟悉，但在製作架構圖時，透過圖像化的方式，加上使用 ChatGPT 協助我們整理概念與說明內容，讓我們能更清楚理解各層的功能與關係，也幫助我們把複雜的內容轉換成比較容易說明的文字。

在整個創作過程中，最大的挑戰是如何把專業又複雜的系統內容，用簡單清楚的方式呈現出來。一開始做出的內容不夠清楚，圖片與說明也需要不斷修正，但透過反覆討論、修改，以及善用 ChatGPT 作為學習輔助工具，我們逐步完成了一份結構完整的報告內容。
